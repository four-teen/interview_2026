Program Chair Module User Manual

This manual covers the Program Chair module only, starting from login at http://localhost/interview/.

1. Login (Program Chair)

  1. Open http://localhost/interview/.
  2. Click Authorized Google Sign-In.
  3. Sign in with your official SKSU Google account.
  4. If your role is Program Chair and your program is unlocked, you are redirected to progchair/index.php.
Common login notes: non-admin login lock, per-program lock, and inactive account status can block Program Chair access.

2. Dashboard Overview

  • Student workspace (search, qualified list, scrolling cards).
  • Cutoff info (global SAT range or program SAT cutoff).
  • Program cards (cutoff, scored count, capacity).
  • Sidebar action cards: Pending Transfers, Unscored, Needs Review, Activity Logs.

3. Main Workflow

3.1 Find Student

  1. Use top search bar (examinee number or name).
  2. Click available action on the student card.
  • Manage: no interview yet.
  • Manage Details: interview exists and you are owner.
  • View Details: interview exists but not owned by your account.
  • Accept/Reject: pending transfer for your program.

3.2 Create/Update Interview Details

  1. Open Manage or Manage Details.
  2. Fill required fields: classification, ETG class (if ETG), mobile number, SHS track, 2nd and 3rd choice.
  3. Click Save.
  • 1st choice is forced to the assigned Program Chair program.
  • Only owner Program Chair can update the record.
  • First save auto-provisions student portal credentials.

3.3 Enter Interview Scores

  1. Click Enter Scores from student details modal.
  2. Input component scores and save.
  • SAT score is auto-loaded and not editable.
  • Final score is computed server-side.
  • Only owner Program Chair can save/update scores.
  • Actions are recorded in Activity Logs.

3.4 Transfer Student

  1. Click Transfer on a student card.
  2. Review summary on transfer page.
  3. Select target program and submit request.
  • Only same-campus programs are listed.
  • Cannot create a new transfer while one is pending.
  • Global SAT cutoff override applies when active.

3.5 Approve/Reject Incoming Transfers

  1. Open Pending Transfers.
  2. Review request details.
  3. Approve or Reject.

On approval, interview ownership and assigned program move to the receiving Program Chair/program.

3.6 Ranking, SCC, and Export

  1. Click a program card in right panel.
  2. In ranking modal, review list, add SCC (Regular), print, or export Excel.
  • SCC accepts only REGULAR students.
  • SCC follows endorsement capacity.
  • Students already covered by regular ranking cannot be added through SCC.

3.7 My Actions Queue

  • Pending Transfers: incoming transfer approvals.
  • Unscored: owned interviews with no final score yet.
  • Needs Review: owned scored interviews without pending transfer.
Recommended daily order: 1) Pending Transfers, 2) Unscored, 3) Needs Review.

3.8 Activity Logs

  1. Open Activity Logs.
  2. Review trend chart and table.
  3. Track SCORE_SAVE, SCORE_UPDATE, FINAL_SCORE_UPDATE events.

5. Quick Troubleshooting

  • Cannot login: check account status, non-admin lock, and your program lock.
  • Student not visible: verify latest batch and cutoff configuration.
  • Cannot edit details: only interview owner can edit.
  • Cannot transfer: resolve existing pending transfer first.
  • Cannot add SCC: student must be REGULAR and SCC capacity available.
